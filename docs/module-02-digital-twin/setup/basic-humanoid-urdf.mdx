# Basic Humanoid Robot URDF Model

This document provides a basic URDF model for a humanoid robot that works in both Gazebo and Unity for the Digital Twin module.

## URDF Model Overview

The following URDF model defines a simplified humanoid robot with:
- Single body (torso)
- Head
- Two arms with simple geometry
- Two legs with simple geometry
- Basic joint constraints

## Launch File for Testing

Create a launch file `test_humanoid.launch.py`:

```python
from launch import LaunchDescription
from launch.actions import DeclareLaunchArgument
from launch.substitutions import Command, LaunchConfiguration
from launch_ros.actions import Node
from launch_ros.substitutions import FindPackageShare
import os

def generate_launch_description():
    # Get the package directory
    pkg_share = FindPackageShare('basic_humanoid_description').find('basic_humanoid_description')

    # URDF path
    urdf_path = os.path.join(pkg_share, 'urdf', 'basic_humanoid.urdf')

    # Robot state publisher node
    robot_state_publisher = Node(
        package='robot_state_publisher',
        executable='robot_state_publisher',
        name='robot_state_publisher',
        output='screen',
        parameters=[{
            'robot_description': Command(['xacro ', urdf_path])
        }]
    )

    # Joint state publisher node
    joint_state_publisher = Node(
        package='joint_state_publisher',
        executable='joint_state_publisher',
        name='joint_state_publisher',
        output='screen'
    )

    return LaunchDescription([
        robot_state_publisher,
        joint_state_publisher
    ])
```

## Testing the URDF Model

### 1. Validate the URDF
```bash
# Check URDF syntax
check_urdf ~/digital_twin_ws/src/basic_humanoid.urdf

# Or use xacro if using xacro format
ros2 run xacro xacro ~/digital_twin_ws/src/basic_humanoid.urdf
```

### 2. Visualize in RViz
```bash
# Source your workspace
source ~/digital_twin_ws/install/setup.bash

# Launch robot state publisher
ros2 launch test_humanoid.launch.py

# In another terminal, launch RViz
rviz2
```

### 3. Load in Gazebo
```bash
# Source your workspace
source ~/digital_twin_ws/install/setup.bash

# Launch Gazebo with the robot
gz sim -r humanoid_test.world

# In another terminal, spawn the robot
ros2 run ros_gz_sim create -file ~/digital_twin_ws/src/basic_humanoid.urdf -name basic_humanoid -allow_renaming true
```

## Unity Compatibility Considerations

For Unity compatibility, the following aspects should be considered:

1. **Coordinate System**: Unity uses a left-handed coordinate system (X-right, Y-up, Z-forward) while ROS uses a right-handed system (X-forward, Y-left, Z-up). The URDF model should account for this when visualized in Unity.

2. **Mesh Simplification**: For real-time performance in Unity, consider using simplified collision geometries (boxes, spheres, cylinders) as used in this model.

3. **Joint Limitations**: Unity visualization should respect the joint limits defined in the URDF.

4. **Material Mapping**: The materials defined in the URDF can be mapped to Unity materials during the import process.
